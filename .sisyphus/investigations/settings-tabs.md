## Notes behind the classifications

- `role-case-bug`: admin mutations for Work Types, Bank Accounts, and Webhooks were previously blocked by `@Roles('ADMIN')`/case drift and are covered by T2.
- `route-order-bug`: no current `/settings` tab uses `PUT /api/settings/bulk`, so T3 does not explain any tab in `SettingsPage.tsx` today.
- `unwrap-bug`: the planned T7 assumption does not match the current backend. `GET /api/users/me` returns a plain object from `UsersController.getProfile()`.
- `dto-mismatch`: General settings use the wrong setting keys entirely (`appName` vs `app.name`, etc.), so both hydration and save target the wrong contract.
